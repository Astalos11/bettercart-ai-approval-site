# Affiliate Application Strategy Update

Date: 2026-05-17

## Decisions

### Brand Name

Use the current brand name:

```text
BetterCart AI
```

Do not hedge public application copy around a possible rename.

### Advertiser Application Strategy

After network approval, apply broadly enough to learn approval rate.

Working approach:

```text
apply to many relevant and adjacent advertisers
-> record approval / rejection / response time
-> inspect real feed / deep-link quality where approved
-> rank usefulness after evidence exists
```

Food / grocery / nutrition / snack / beverage / supplement advertisers remain higher priority, but the project should not over-optimize too early around a tiny hand-picked set.

Reason:

- Approval rate is itself an unknown.
- Advertiser terms and feed fields cannot be inferred reliably from public listings.
- A wider application set gives better evidence about which networks and categories are realistic for MVP.

### Website Quality Bar

The site should not stay frozen at "minimum viable approval site."

New direction:

```text
keep improving toward a credible real content site
```

Allowed improvements:

- more useful guides
- clearer comparison examples
- stronger reviewer pages
- better interaction / demo value
- better evidence and disclosure boundaries
- better application and operations docs

Still prohibited:

- fake reviews
- fake traffic
- fake testimonials
- fake affiliate partnerships
- medical / disease claims
- undisclosed paid placement
- real affiliate links before approval

## Operational Impact

Application tracker should support volume:

- many advertiser rows
- relevance tier
- approval status
- response time
- feed availability
- field coverage
- comparison-policy restrictions
- MVP usefulness after evidence is known
