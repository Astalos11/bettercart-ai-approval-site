## Positioning

Short description:

```text
BetterCart AI is an AI-assisted packaged food comparison guide for US grocery shoppers. The site publishes original food buying guides, sample comparison pages, and nutrition-label education content focused on packaged food shopping decisions.
```

Longer description:

```text
BetterCart AI helps shoppers compare packaged foods such as snacks, protein bars, cereals, pantry foods, beverages, frozen meals, sauces, and condiments. The site focuses on nutrition facts, ingredient context, serving size, category tradeoffs, and shopping intent. Current content includes buying guides, sample comparison pages, disclosure pages, editorial standards, a publisher kit, a searchable site index, and a static USDA-derived comparison demo.
```

Promotional methods:

```text
Editorial shopping guides, packaged food comparison pages, nutrition-label education content, searchable topic pages, and future AI-assisted product discovery tools. Affiliate links, if added, will be clearly disclosed and used only where they fit the reader's shopping context and program rules.
```

Traffic / history note:

```text
The site is newly launched. Current focus is building original editorial content, reviewer-friendly policy pages, and responsible AI-assisted comparison workflows. We are not claiming established traffic or prior affiliate sales history.
```

## Claims To Avoid

Do not claim:

- existing affiliate partnerships
- existing sales history
- established traffic volume
- live price comparison
- complete product coverage
- medical recommendations
- product testing that did not happen
- guaranteed positive coverage
- universal healthy / unhealthy food grades

## Network Approval Checklist

Before submitting:

- Live URL opens from a private browser.
- `/about/`, `/contact/`, `/affiliate-disclosure/`, `/privacy/`, `/for-reviewers/` all open.
- `contact@bettercartai.com` can receive external email.
- `robots.txt` and `sitemap.xml` are live.
- Homepage does not show under-construction language.
- Application description matches the site content.

## After Awin Approval

Do not immediately build an ingestion pipeline.

First record:

- approved account id / publisher id
- whether payment / tax setup is required before advertiser applications
- advertiser search categories available for food, grocery, health, supplement, beverage, pantry
- whether legacy ShareASale programs appear inside Awin

Then apply to 10-20 relevant advertisers and track:

- advertiser name
- advertiser id
- category
- approval status
- auto-approved or manual review
- response time
- rejection reason if any
- product feed available
- product count
- feed fields visible
- UPC / GTIN present
- product URL present
- image URL present
- price present
- deep link available
- content restrictions
- negative comparison / review restrictions

## MVP Decision Rule

Do not build generalized feed import until at least:

```text
3 real approved commerce surfaces
```

have been inspected.

Better target:

```text
8-15 useful advertisers / merchants
```

Useful means:

- relevant to packaged food / grocery / supplement / beverage / pantry
- has stable links or product feeds
- terms allow editorial shopping-guide use
- fields can support product identity or manual enrichment

## Fallback If Declined

If declined:

1. Save the exact rejection text.
2. Check whether the issue is website ownership, content quality, description clarity, or traffic source.
3. Update the reviewer page or publisher kit only if there is a clear gap.
4. Apply to FlexOffers / Skimlinks / Sovrn-style aggregator in parallel.
5. Continue UPC / public product identity experiments without claiming affiliate relationships.
